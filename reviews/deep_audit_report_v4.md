## 四、生成完全可用的视频，还缺什么？

### 最核心的缺口（按严重度）

**缺口 1：Linux 服务器部署字幕不可用（P0）**
- Docker/Linux 默认无中文字体 → 字幕豆腐块 → 视频废的
- 修复成本：极低（加字体安装说明 + 字体缺失时明确报错 + fallback 下载机制）

**缺口 2：网络可靠性不足（P0/P1，v3 遗留）**
- POST 不重试 + 轮询无网络重试 + 下载无重试 + 429 不重试 → 网络波动直接报废整次生成
- 长耗时任务（几分钟到十几分钟）因瞬断失败，用户体验极差
- 修复成本：低（加统一重试装饰器）

**缺口 3：关键词高亮功能是死代码（P0）**
- 用户以为有高亮，实际完全不生效
- 默认 pop 动画和口播 typewriter 动画都没处理 highlight
- 修复成本：低（在 pop/typewriter 中加高亮标签）

**缺口 4：config.example.py 误导新用户（P0，v3 遗留）**
- 注释说"已迁移 Omni"但值还是 text2video → 新用户按 example 配完直接 API 404
- 还缺 KLING_ACCESS_KEY / KLING_SECRET_KEY / TRANSITION_STYLE / accent_color 等字段
- 修复成本：极低

**缺口 5：核心逻辑无测试（P1）**
- 1107 行主函数零测试 → 每次改动都是赌博
- 错误处理、降级路径、合规拦截全靠人肉验证
- 修复成本：高（需要大量测试用例编写）
