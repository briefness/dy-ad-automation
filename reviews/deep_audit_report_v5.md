## 四、生成完全可用的视频，还缺什么？

### 最核心的质量缺口（影响输出质量）

**缺口 1：无 Negative Prompt（P0）**
- 对面部崩坏、多余肢体、水印文字等常见 AI 视频缺陷零防御
- 修复成本：极低（把 CHARACTER_NEGATIVE_PROMPT 或通用视频 negative prompt 传给 generate_video）

**缺口 2：无角色一致性校验（P1）**
- 生成了但不知道角色崩没崩、换没换人
- 没有检测 → 无法做针对性优化 → 输出质量不可控
- 修复成本：中（需要人脸相似度比对算法或 API）

**缺口 3：参考图过载稀释角色权重（P1）**
- 5 张参考图反而可能比 3 张效果差
- 串行模式下场景锚点 + 前帧尾 + 角色 + 产品 信息冗余
- 修复成本：低（精简到 3 张，或动态调整参考图策略）

### 最影响使用体验的缺口

**缺口 4：无断点续跑（P1）**
- 生成到第 4 段失败了，前 3 段白花钱，得从头来
- 长视频（60s）+ 高失败率场景下体验极差
- 修复成本：中（需要 checkpoint 持久化 + 重启时检测已有片段）

**缺口 5：无幂等性/缓存（P1）**
- 同样的输入反复生成反复花钱
- 调参/测试场景下浪费严重
- 修复成本：中（需要内容哈希 + 结果缓存）

## 五、优化方向（按性价比排序）

### 超高性价比（< 30 分钟，质量/体验立竿见影）

| 优化项 | 改动量 | 收益 |
|--------|--------|------|
| 视频生成加 negative_prompt | 5 行 | 大幅减少面部崩坏/多余肢体/水印 |
| 精简参考图到 3 张以内 | 10 行 | 角色保真度可能反而提升 |
| 一致性描述移到 prompt 开头 | 5 行 | 一致性指令权重提升 |
| base_prompt 光影描述去重 | 10 行 | 消除光影指令冲突 |
| POST 请求加重试 | 10 行 | 网络瞬断失败减少 70%+ |

### 中性价比（几小时，明显提升）

| 优化项 | 改动量 | 收益 |
|--------|--------|------|
| 轮询 + 下载加网络重试 | 30 行 | 长任务抗网络波动 |
| 片段级缓存（按 prompt+seed 哈希） | 50 行 | 重复生成省配额 |
| 失败保留中间文件 + --resume | 80 行 | 断点续跑，不浪费已生成内容 |
| 参考图加文字描述（角色/场景/产品） | 10 行 | 模型更清楚每张图的参考重点 |
| 模板系统补全参数覆盖 | 20 行 | 模板真正可复用 |

### 低性价比 / YAGNI 候选

| 优化项 | 判断 |
|--------|------|
| AI 质量评分接入 | 看需求：传统 CV 粗筛 + 人工验收也够用 |
| 跨片段人脸相似度校验 | 需要额外模型/API，成本高 |
| 多平台适配（快手/小红书/B站） | YAGNI：先把抖音做深 |
| A/B 对比工具 | YAGNI：用户自己看两个视频对比也行 |
| 产品预设扩充到子分类级 | 看业务需求，11 大类 + default 够起步 |
| 上帝函数拆分 | 迟早要做，但不是当前质量瓶颈 |
| 测试体系建设 | 长期工程，优先级看团队规模 |
